3. The Product Class

This is the foundation of everything. It defines what a "Product" actually is in our code.

What's a Class? A class is like a blueprint or template for creating objects. Think of it as a form that describes what data and actions a thing should have. The Product class describes what information every product in our store should have (name, price, cost, etc.).

Action: Create the Class

  • Right-click Source Packages → New → Java Class
  • Name: Product
  • Package: model

Action: The Code

ðŸ’¡ Pro Tip: Before you type ANY code, write comments describing what each section does. This helps you think through the logic first!

Copy this logic into your file. Note how we use private variables for encapsulation.

package model;

public class Product {

    // 1. Instance variables (The Data)
    private String name;  // String = text
    private String description;  // String = text
    private double price;  // double = decimal number
    private double cost;  // double = decimal number

    // 2. Constructor (How to build a Product)
    public Product(String name, String description, double price, double cost) {
        this.name = name;
        this.description = description;
        this.price = price;
        this.cost = cost;
    }

    // 3. Getters (Allowing others to read the data)
    public String getName() { return name; }
    public String getDescription() { return description; }
    public double getPrice() { return price; }
    public double getCost() { return cost; }

    // 4. toString (Important for displaying in lists!)
    @Override
    public String toString() {
        return name + " ($" + price + ")";
    }
}

4. The Warehouse

The Warehouse acts as the storage facility. It holds a list of all your Products.

What's an ArrayList? An ArrayList is a container that holds multiple items and can grow dynamically. Think of it like a flexible shopping bag that expands as you add more products. Unlike a fixed array, it automatically resizes itself.

Action: Create the Class

  • Right-click model package → New Java Class
  • Name: Warehouse

Action: The Code

ðŸ’¡ Pro Tip: Start by writing comments for what the Warehouse should do, then fill in the code beneath each comment.

We need an ArrayList to make our storage dynamic (expandable).

package model;
import java.util.ArrayList; // Don't forget this import!

public class Warehouse {

    // The list that holds our Product objects
    private ArrayList<Product> products;

    public Warehouse() {
        products = new ArrayList<>();
    }

    public void addProduct(Product product) {
        products.add(product);
    }

    public ArrayList<Product> getProducts() {
        return products;
    }
}

private Warehouse warehouse;

private double currentTotal = 0.0;

public ShopFrame(Warehouse w) {
    initComponents();
    this.warehouse = w; 
    
    // Setup list
    model = new DefaultListModel<>();
    lstShopItems.setModel(model);

    for (Product p : warehouse.getProducts()) {
        model.addElement(p.toString());
    }
}

ðŸ“š Java Concepts Reference

Here's a quick guide to all the key Java concepts used in this tutorial:

Object-Oriented Programming (OOP)

Class: A blueprint that defines what data and methods an object should have. Classes are used to create specific instances (objects) that follow that blueprint.
Object: An actual instance of a class. For example, if Product is the class (blueprint), then a specific product like "Laptop ($999)" is an object.
Encapsulation: Hiding internal details and only exposing what's necessary. We use private variables with public getter methods so other code can't accidentally break our data.
Constructor: A special method that runs when you create a new object. It's like the setup instructions. The method name matches the class name (e.g., Product(...)) and takes parameters to initialize the object.

Data Types

String: Text data. Examples: "Hello", "Product Name". Always in quotes.
double: A decimal number (number with decimals). Examples: 19.99, 5.0. Use this for prices!
int: A whole number (no decimals). Examples: 42, -5, 0.
boolean: True or false. Used for yes/no or on/off conditions.

Collections & Data Structures

ArrayList: A flexible container that holds multiple objects and automatically grows or shrinks. You access items by their index (position). Syntax: ArrayList<Type> means "an ArrayList that holds Type objects."
DefaultListModel: A Swing component that manages what appears in a GUI list. When you add items to it, they automatically show up on screen.

Methods & Access

public: Anyone can access this. Use for methods you want other classes to call.
private: Only this class can access this. Use for internal data to prevent accidents.
Getter: A public method that returns private data. Example: getName() lets others read the name without modifying it.
void: A method that does something but doesn't return a value. Example: addProduct() adds to the warehouse but doesn't return anything.
@Override: Tells Java that we're replacing a method from the parent class. For example, toString() is being customized.

Common Patterns

Dependency Injection: Passing objects into constructors instead of creating them inside. This lets different classes share the same object. Example: passing warehouse into WarehouseFrame so it can access the same data.
Model-View-Controller (MVC): Separating code into three parts: Model (data classes like Product), View (GUI like MainMenu), and Controller (logic that connects them).
toString(): A special method that defines how an object appears when printed or shown in a list. Make it user-friendly!

GUI & Events

JFrame: A window. The top-level container for GUI components.
JList: A GUI component that displays a list of items to the user.
TextField: A GUI component where users can type a single line of text.
Button (ActionListener): When a user clicks a button, an event is triggered. You write code in the action listener to respond to that click.
