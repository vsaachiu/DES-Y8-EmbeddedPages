1. Project Setup

We are going to build a "Shop System" where one central file manages your data (products/orders) and your windows (GUI).

In NetBeans:
  1. File → New Project → Java Application.
  2. Project Name: Use your name + Shop (e.g., JohnDoeShop).
  3. Create Main Class: ‚úî CHECK THIS BOX.
  4. Click Finish.
Why? The file JohnDoeShop.java will become our "Manager". It will hold the lists of products and orders, and it will be responsible for launching the windows.

2. The Product Class

This defines what an item in our store looks like.

Action: Create the Class

  • Right-click Source Packages → New → Java Class.
  • Name: Product.

Action: The Code

public class Product {

    // 1. Data
    private String name;
    private double price;

    // 2. Constructor
    public Product(String name, double price) {
        this.name = name;
        this.price = price;
    }

    // 3. Getters
    public String getName() { return name; }
    public double getPrice() { return price; }

    // 4. Display
    @Override
    public String toString() {
        return name + " ($" + price + ")";
    }
}

3. The Order Class

This defines a customer purchase. An order is basically a list of products that someone bought.

Action: Create the Class

  • Right-click Source Packages → New → Java Class.
  • Name: Order.

Action: The Code

import java.util.ArrayList;

public class Order {

    private ArrayList<Product> items;
    private double total = 0;

    public Order() {
        items = new ArrayList<>();
    }

    public void addItem(Product p) {
        items.add(p);
        total = total + p.getPrice();
    }

    public String toString() {
        return "Order ID: " + items.hashCode() + " - Total: $" + total;
    }
}

4. The Manager (Main Class)

This is the most important step. Your main file (e.g., JohnDoeShop.java) will hold everything together.

Action: Open your Main Class

Open the file created in Step 1. We will add our Lists (Data) and our Windows (GUI) here.

import java.util.ArrayList;

public class JohnDoeShop {

    // --- 1. THE DATA ---
    public static ArrayList<Product> inventory = new ArrayList<>();
    public static ArrayList<Order> orders = new ArrayList<>();

    // --- 2. THE WINDOWS ---
    // We make them static so we can access them easily
    public static MainMenu menuWindow;
    public static WarehouseFrame warehouseWindow;
    public static ShopFrame shopWindow;
    public static OrdersFrame ordersWindow;

    // --- 3. THE START BUTTON ---
    public static void main(String[] args) {
        
        // Create the windows
        menuWindow = new MainMenu();
        warehouseWindow = new WarehouseFrame();
        shopWindow = new ShopFrame();
        ordersWindow = new OrdersFrame();

        // Show the menu to start
        menuWindow.setVisible(true);
    }
}
You will see RED ERROR LINES because we haven't created the Window files (Frames) yet. That is normal! Proceed to the next step to fix them.

5. UI: Warehouse Window

This window lets you add items to the inventory list inside your Manager class.

Action: Setup

  1. Right-click Package → New → JFrame Form.
  2. Name: WarehouseFrame.
  3. Design: Add TextFields (Name, Price) and a Button ("Add Stock").

Action: The Logic

Double-click the "Add Stock" button.

private void btnAddStockActionPerformed(java.awt.event.ActionEvent evt) {
    
    // 1. Get Data
    String name = txtName.getText();
    double price = Double.parseDouble(txtPrice.getText());

    // 2. Create Product
    Product p = new Product(name, price);

    // 3. Add to the Manager's Inventory
    JohnDoeShop.inventory.add(p);

    // 4. Feedback
    lblStatus.setText("Added " + name);
}

6. UI: Shop Window

This window reads the inventory and creates orders.

Action: Setup

  1. Right-click Package → New → JFrame Form.
  2. Name: ShopFrame.
  3. Design: Add a JList (inventory), a Button ("Add to Cart"), and a Button ("Checkout").

Action: Refresh Data

When this window opens, we need to see the latest products. Go to Source and add a method:

public void refreshList() {
    DefaultListModel m = new DefaultListModel();
    
    // Loop through the Manager's inventory
    for(Product p : JohnDoeShop.inventory) {
        m.addElement(p);
    }
    
    myJList.setModel(m);
}

üìö Vocabulary

Static: A keyword that means "Belongs to the class, not a specific object." We used static in our Main class so that JohnDoeShop.inventory is a single, shared list that everyone can access easily.
State Management: The practice of deciding where your data lives. In this project, the "State" (Inventory and Orders) lives in the Main Class, and the Windows just "view" or "modify" that state.
Instance vs. Class:
Class: The blueprint (e.g., Product.java).
Instance: The specific object (e.g., Product p = new Product(...)).
Method Call: Triggering an action. When we type refreshList(), we are calling a method we wrote to update the screen.
